Addon Modules

addon/ フォルダにPythonモジュールを配置すると、 起動時に自動で読み込まれるアドオンとして動作します。

1. 追加の基本方針

  1. VillageCore-main/addon/addons/フォルダ内に .pyファイルを作成します。
  2. DiscordのCogを作成します。
  3. async def setup(bot) を定義してCogを追加します。
  4. ボットを再起動すると自動的に読み込まれます。

2. 追加手順

Step 1: ファイルを作る

例: 

addon/my_addon.py

Step 2: コードを書く 

import discord
from discord.ext import commands


class MyAddon(commands.Cog):

    def __init__(self, bot: commands.Bot):
        self.bot = bot


    @commands.Cog.listener()
    async def on_ready(self):
        print("MyAddon loaded")


async def setup(bot: commands.Bot):
    await bot.add_cog(MyAddon(bot))

Step 3: スラッシュコマンド追加 

@app_commands.command(
name="myaddon_ping",
description="テスト用コマンド"
)

async def myaddon_ping(
self,
interaction: discord.Interaction
):

    await interaction.response.send_message(
        "Hello from addon",
        ephemeral=True
    )

Step 4: ボットを再起動 

python bot.py

成功すると以下のようなログが表示されます。 

📦addons loaded successfully: 読み込まれているアドオン数

📦loaded addon list: 読み込まれているアドオン名

3. 追加できる内容

  • スラッシュコマンド
  • 通常コマンド
  • イベントリスナー
  • DBを使った記録処理
  • 独自ロジック

4. 読み込み無効化

DISABLED_ADDONS 環境変数で無効化できます。 

$env:DISABLED_ADDONS="addon.sample_addon"

複数指定: 

$env:DISABLED_ADDONS="addon.sample_addon,addon.my_addon"

5. よくあるミス

  • setup(bot)を書き忘れる
  • ファイル名とimport名が違う
  • addonフォルダ外に作成する
  • discord.pyのimportミス
  • 再起動していない

6. 追加後の確認

  1. 起動ログを確認
  2. Discordでコマンド確認
  3. 動作確認

7. テンプレート 

import discord
from discord import app_commands
from discord.ext import commands


class TemplateAddonCog(commands.Cog):
    def __init__(self, bot: commands.Bot):
        self.bot = bot

    @app_commands.command(name="template_demo", description="テンプレート用のサンプルコマンド")
    async def template_demo(self, interaction: discord.Interaction) -> None:
        await interaction.response.send_message("Template addon loaded", ephemeral=True)


async def setup(bot: commands.Bot) -> None:
    await bot.add_cog(TemplateAddonCog(bot))
